# Troubleshooting: Empty expression when running custom XSLT

In the XML map editor, a custom XSLT transform has a parameters
table in the properties page that shows the name and value pair of
the parameters.  By default, the value of a parameter is empty, which
can cause the empty expression error when you test the map. In the
properties view of the custom XSLT transform, enter the name of the
input element as a value in the parameter column and save your map
to generate the xslt.
