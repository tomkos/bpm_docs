Defining and modifying project areas by using the command line

About this task

Each development environment requires a default project area that is named dev_env_connection_definition. However, you can define more project areas as needed. Because you might have multiple people working in the development environment on multiple projects, you can define project areas to limit the effects of resetting the test environment. When you reset the test environment from Case Builder, only a single project area is reset. The entire object store is not affected, and other users can continue working without interruption.
