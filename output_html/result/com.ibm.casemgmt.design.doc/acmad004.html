Restoring workflow components

You can restore Business Automation Workflow components by using your latest backups. Work with your backup administrator to understand the backup policy and the best way to perform the restore.

Procedure

To restore the workflow components:

  • Restore the workflow servers.
    1. Reinstall the Business Automation Workflow software as needed.
    2. Reconfigure the software or restore the workflow configuration files from a backup.
  • Restore the Business Automation Workflow network share that hosts runtime plug-ins, add-ons configuration, documents, and customized widget pages (run time) from the latest backup.
  • Restore the case management solutions (design time or run time) and customized widget pages (design time) by recovering the object store. See Recovering the data in your FileNet P8 domain.
  • Optional: Restore the case history store database from the database server.
  • Optional: Restore the business rules repository. Restore and overwrite the network shared directory from the latest backup.
