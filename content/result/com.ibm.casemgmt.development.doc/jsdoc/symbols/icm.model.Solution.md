The object-valued properties that are supported are properties containing
custom dependent objects.  Besides a data type of "xs:object" these properties have a cardinality of "LIST".
In addition they are properties defined by an application, not the system.
The value of such a property holds an array containing instances of icm.model.DependentObject.

In addition to the dependent object classes corresponding to the attribute definitions of the solution,
this array might be populated when retrieveDependentObjectClasses() is called.  Dependent object
classes that are not referenced by any active attribute definition
in the solution can be independently retrieved, although this is not typically the case.
