# When to choose containers

Deploying products in a container environment can be faster and easier than a standard
traditional installation. Additionally, updating software in a container environment can be simpler
and less disruptive. But there are also reasons to maintain a traditional installation model.
Consider the relevant factors when deciding whether to use containers. For information about the
differences between the traditional and container environments of Business Automation Workflow, see Runtime environments.

- Dynamic scalability to adjust to workload
- Zero downtime support for patching
- Ability to not only quickly patch or upgrade but quickly rollback to previous versions
- Better resiliency for your applications
- Lower cost for application isolation

- You use deprecated capabilities or artifacts.
- Your applications contain advanced content that is developed in IBM Integration
Designer.
- You use custom applications that use EJBs.
