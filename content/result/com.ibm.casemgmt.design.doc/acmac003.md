# generateConfig command

## Syntax

```
configmgr\_cl generateConfig
 [-box ]
 -deploy deploy\_type
 -profileType profile\_type -task task\_type
 -taskfile task\_file\_name -taskname task\_display\_name
 -profile myprofile 
  [-silent] [-force] [-help]
```

- The name of the profile, such as develop1. The profile is located in the
install\_root/CaseManagement/configure/profiles directory.
install\_root is the location where IBM Business Automation

- The full path to the profile directory, such as
"install\_root\CaseManagement\configure\profiles\develop1" or
/install\_root/CaseManagement/configure/profiles/develop1.
- The full path to the profile input file, such as
"install\_root\CaseManagement\configure\profiles\develop1\develop1.cfgp"

/install\_root/CaseManagement/configure/profiles/develop1/develop1.cfgp.

## Sample commands

```
configmgr\_cl generateConfig 
 -deploy standard
 -profileType devenv
 -profile Develop1
```

```
configmgr\_cl generateConfig 
 -profileType prodenv
 -task deploysolution -profile Production1
```

If
a deploysolution.xml file is already in the profile directory, the file name is
incremented, for example, deploysolution.2.xml.

```
configmgr\_cl generateConfig 
 -profileType prodenv
 -task deploysolution  -profile ObjectStore2
 -taskname "Deploy to object store ObjectStore2"
```

```
configmgr\_cl generateConfig -help
```
